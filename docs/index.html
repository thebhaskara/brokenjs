API Documentation

Model
class source code
description
Model is structure that can handle data, with typical features like get, set and watch, also with many additional features.
Following are all the features that this stucture offers...
Features
Attributes
feature source code
description
initializes internal attributes object by the object provided, otherwise an empty object is initialized.
parameters
{String} [attributes] - initial attributes object.
example
var attributesObj = new Attributes();
attributesObj.set('employee', { id:10, name: 'Ryan' })
attributesObj.get('employee.name'); // returns 'Rayn'
attributesObj.get('employee.id'); // returns 10
functions
get
function source code
description
Get value from attributes, at the path provided.
Internally uses lodash _.get
parameters
{String} [path] - string object path.
example
attributesObj.get('employee.name'); // returns 'Rayn'
attributesObj.get('employee.id'); // returns 10
set
function source code
description
Set value to attributes, at the path provided.
Internally uses lodash _.set
parameters
{String} path - string object path.
{Object} value - value to be stored at the provided path.
example
attributesObj.set('employee', { id:10, name: 'Ryan' })
destroy
function source code
description
Deletes internal attributes instance.
example
attributesObj.destroy();
Destroy
feature source code
description
This feature adds the ability to call the appropriate callbacks before destorying the instance.
This provides onDestroy(Callbacks) property in the instance.
This is designed to be used to do necessary actions before destorying, such as preventing memory leaks, unwatching, detaching events, etc...
functions
destroy
function source code
description
Calls onDestroy callbacks and then deletes onDestroy object
example
var destoryableObj = new Destory();
destoryableObj.onDestroy.add(() => console.log('called on destroy'));
destoryableObj.destroy(); // logs 'called on destroy' on console
Forward
feature source code
description
This class provides the ability to forward values that are set on source model object to destination model object

Injector
feature source code
description
This feature adds the ability to pass values at paths into a given function. The properties that are passed in the format shown in the example, will be replaced with a function that has these paths injected as parameters into it. Observe that prop1 is being used as a callback when called from prop2.
example
let mergeable = {
prop1: ['path1', 'path2', function(path1Value, path2Value){
// your code here
}],
prop2: function(){
this.prop1();
}
}
let merged = Broken.Merge(mergeable);
let mergedObj = new merged();
mergedObj.prop1();
Watcher
feature source code
description
This feature adds the ability to listen to one path at a time. This works only if Attributes feature is included.
functions
watch
function source code
description
watches all the paths provided and passes the values of those paths to the callback everytime a set is triggered on any of those paths.
parameters
{Watcher|Model|ViewModel} [component] - source component to watch, if not provided, watches itself.
{String} path - attribute's path.
{Function} callback - callback to be triggered.
example
watcherObj.watch('path', function(pathValue){
// your code here
})
unwatch
function source code
description
unwatches a watch.
parameters
{Number} watchId - Id of the watch to be unwatched.
example
let watchId = watcherObj.watch('path1', (path1Value) => console.log(path1Value));
watcherObj.set('path1', 10); // prints 10 on console.
watcherObj.unwatch(watchId);
watcherObj.set('path1', 10); // does not print 10 on console.
set
function source code
description
triggers a watch.
parameters
{String} [path] - Path to be triggered
example
let watchId = watcherObj.watch('path1', (path1Value) => console.log(path1Value));
watcherObj.set('path1', 10); // prints 10 on console.
watcherObj.unwatch(watchId);
watcherObj.set('path1', 10); // does not print 10 on console.
destroy
function source code
description
unwatches all external watches and deletes own watches.
parameters
{String} [path] - Path to be triggered
example
watcherObj.destroy()

feature source code
description
This feature adds the ability to listen to a list of paths

watchAll
function source code
description
watches all the paths provided and passes the values of those paths to the callback everytime a set is triggered on any of those paths.
parameters
{String} path - first path.
...{String} [path] - any number of paths.

})

getParent
function source code
description
Gets parent. If a name is passed, gets the closest ("grand")parent with that name.
passing name helps when components are being wrapped
parameters
{String} [name] - name of the parent required.
example
parentObj.getParent(); // gets immediate parent
parentObj.getParent('name'); // gets closest parent with name

Initializer
feature source code
description
This feature runs all the functions stating with "initialize_" in the constructor (while creating a new object)
example
let Fact = {
initialize_message: function(attributes, options){
console.log('initialize_message method is called!')
}
}
let Factory = Broken.Merge(Fact);
let obj = new Factory(); // prints 'initialize_message method is called!' message
Callbacks
class source code
description
Callbacks is structure that can handle registering callbacks from multiple places and then trigger from one place.
example
function fn1( value ) {
console.log( value );
}

function fn2( value ) {
console.log( "fn2 says: " + value );
return false;
}

let callbacks = new Callbacks();
callbacks.add(fn1);
// Outputs: foo!
callbacks.run( "foo!" );

callbacks.add(fn2);
// Outputs: bar!, fn2 says: bar!
callbacks.run( "bar!" );
functions
add
function source code
description
Adds a callback to the list.
parameters
{Function} callback - callback function to be added
example
callbacks.add(function(){
// added function
});
addWithKey
function source code
description
Adds a callback to the list by key.
parameters
{String} key - key to refer the callback
{Function} callback - callback function to be added
example
callbacks.addWithKey('key1', function(){
// added function
});
remove
function source code
description
removes callback from the list.
parameters
{Function|String} callback|key - key or callback to be removed from the list
example
callbacks.remove(fn1);
callbacks.remove('key1');
run
function source code
description
removes callback from the list.
parameters
{arguments} arguments - set of arguments to be passed to the callbacks
example
callbacks.run('v1', 'v2');
runWith
function source code
description
removes callback from the list.
parameters
{Object} context - context to set while running callbacks
{Array} args - arguments as an array to pass to the callbacks
example
callbacks.runWith({ a: 10 }, [ 20, 30 ]);
disable
function source code
description
Disables the callbacks. Once disabled, run can't trigger callbacks.
example
callbacks.disable();
enable
function source code
description
Enables the callbacks. Once enabled, run can trigger callbacks. Callbacks is enabled when initialized.
example
callbacks.enable();
